# Implementation Spec ‚Äî Attachments (parity P0)

> Generated by the post-fix re-audit. Authoritative worklist for the next dev pass.

## State

- Prisma `Attachment` model EXISTS (id, ticketId?, postId?, fileName, mimeType, size, sha1, storageKey). `Ticket.hasAttachments` + relations exist; `getTicket` already includes attachments. **No migration needed.**
- `FileUploadZone` exists (staff reply `ticket-detail-content.tsx`, client `submit-form.tsx`) but calls a FAKE `simulateUpload()` ‚Äî no HTTP. Parents discard files (`const [, setAttachedFiles]`).
- No upload/download endpoint; reply/create DTOs lack `attachmentIds`. `inbound.service.ts` parses `parsed.attachments[]` but ignores them.

## Storage decision

Local-disk volume now, behind a `StorageService` interface (swap to MinIO/S3 later). Path `/app/uploads` ‚Üí named docker volume. Env: `TELECOM_HD_UPLOAD_DIR=/app/uploads`, `TELECOM_HD_UPLOAD_MAX_SIZE_MB=25` (add to configuration.ts Zod + .env.example).
